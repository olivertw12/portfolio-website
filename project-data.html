Confidentiality Notice

Specific datasets, statistical outcomes, and internal visualizations have been redacted from this case study in compliance with a Non-Disclosure Agreement (NDA). This page focuses on the technical pipeline, code structure, and analytical methodology.

The Challenge: The DMARC Food Pantry Network generates a massive dataset of transactional visitor logs. The data was unstructured and lacked the demographic context needed to make informed decisions about resource allocation.

The Goal: To build a reproducible data pipeline that cleans raw logs, enriches them with ACS Census demographics, and utilizes predictive modeling to identify future service gaps without exposing individual user data.

01

Standardization Pipeline

The foundation of the project was a robust cleaning script designed to handle inconsistencies across 90+ pantry locations. We focused on making the data "universal" for downstream analysis.

Key Transformation Steps

  • MUTE Converted "Date Visited" to Age integers.
  • ENRICH Calculated Income Brackets relative to Federal Poverty Levels.
  • DROP Removed high-variance columns (Housing Type) to reduce noise.
clean_dmarc_data.R 

# Converting Birthdate to Age for Trend Analysis
df <- df %>%
  mutate(Age = as.numeric(difftime(Sys.Date(), DOB, units = "weeks")) / 52.25) %>%
  filter(Age > 0 & Age < 110) # Remove outliers

# Create Income Brackets based on Federal Poverty Levels
df <- df %>%
  mutate(Poverty_Level = case_when(
    Income < 15000 ~ "Extremely Low",
    Income >= 15000 & Income < 30000 ~ "Low",
    TRUE ~ "Moderate/High"
  ))
02

Geospatial Merge Strategy

We needed to identify "At-Risk" zones—areas with high poverty density but low resource availability. To do this, we architected a merge between internal DMARC data and external ACS Census data.

By joining datasets on Zip Code keys, we calculated a "Pantry Density" score. This allowed us to mathematically pinpoint specific neighborhoods (redacted) that were under-served despite having high visitor rates.

Forecasting Demographic Shifts

When new data arrived later in the semester, we lost zip code granularity but gained longitude/latitude data. We pivoted to analyzing specific demographic segments over time.

I utilized the `forecast` library in R to train models on 5 years of historical data. The model identified seasonality patterns (winter peaks) and general upward trends for specific sub-groups, allowing DMARC to anticipate staffing needs for the upcoming year.

predictive_modelling.R 

# Time Series Forecasting for 2025
model <- auto.arima(visitors_ts)
forecast_2025 <- forecast(model, h=12)

# Plotting the prediction structure
autoplot(forecast_2025) +
  labs(title="Projected Visitor Volume (Segmented)", 
       y="Monthly Visits")
